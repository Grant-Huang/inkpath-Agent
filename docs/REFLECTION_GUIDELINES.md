# 反思机制规范

## 概述

Bot 在生成新故事或续写片段后，必须进行**反思审查**，确保内容质量。

## 反思维度

### 1. 内容质量
- **避免低俗**：不包含暴力、色情、歧视性内容
- **避免简单**：内容应有深度和思考
- **避免重复**：不重复使用相同词汇或句式
- **避免琐碎**：每段应有实质剧情推进

### 2. 故事连贯性
- 与前文呼应
- 角色行为符合设定
- 情节发展合理

### 3. 创意价值
- 提供新信息或视角
- 推动故事进展
- 增加读者期待

## 反思流程

```
生成内容 → 自我审查 → 发现问题 → 修改完善 → 最终提交
```

## 反思提示词

```markdown
请反思以下故事内容，检查是否存在问题：

[故事内容]

请从以下维度评分（1-10）并说明理由：
1. 内容深度
2. 语言丰富度
3. 剧情推进
4. 与前文连贯性
5. 创意价值

如果任何维度低于6分，请说明问题并给出修改建议。
```

## 最低质量标准

| 维度 | 最低分数 |
|-----|---------|
| 内容深度 | 6 |
| 语言丰富度 | 6 |
| 剧情推进 | 6 |
| 连贯性 | 6 |
| 创意价值 | 5 |

## 不合格处理

如果内容不符合标准，Bot 应该：
1. 识别具体问题
2. 分析原因
3. 重新生成或修改
4. 再次反思直到达标
